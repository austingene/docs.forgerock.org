038/**
039 * An operation result code as defined in RFC 4511 section 4.1.9 is used to
040 * indicate the final status of an operation. If a server detects multiple
041 * errors for an operation, only one result code is returned. The server should
042 * return the result code that best indicates the nature of the error
043 * encountered. Servers may return substituted result codes to prevent
044 * unauthorized disclosures.
045 *
046 * @see <a href="http://tools.ietf.org/html/rfc4511#section-4.1.9">RFC 4511 -
047 *      Lightweight Directory Access Protocol (LDAP): The Protocol </a>

693    /**
694     * Returns the result code having the specified integer value as defined in
695     * RFC 4511 section 4.1.9. If there is no result code associated with the
696     * specified integer value then a temporary result code is automatically
697     * created in order to handle cases where unexpected result codes are
698     * returned from the server.
699     *
700     * @param intValue
701     *            The integer value of the result code.
702     * @return The result code.
703     */
704    public static ResultCode valueOf(final int intValue) {
705        ResultCode resultCode = null;
706
707        if (intValue >= 0 || intValue < ELEMENTS.length) {
708            resultCode = ELEMENTS[intValue];
709        }
710
711        if (resultCode == null) {
712            resultCode =
713                    new ResultCode(intValue, LocalizableMessage.raw("undefined(" + intValue + ")"),
714                            true);
715        }
716
717        return resultCode;
718    }
